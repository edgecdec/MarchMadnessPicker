# Bugs — Fix these BEFORE working on any PLAN.md tasks

- **Research and implement auto-update for game results from ESPN**: Currently results are only synced when an admin clicks "Sync". Research ESPN's public API rate limits (the scoreboard endpoint at site.api.espn.com). Then design the best approach for auto-updating results when games finish. Options to consider: (1) A lightweight server-side interval (e.g. check ESPN every 60s during game hours, store results if changed) — but consider server resource usage. (2) Client-triggered: when a user visits the bracket/leaderboard page, check if results are stale (>60s old) and trigger a sync — but only one request at a time (debounce across users). (3) A hybrid: server checks every 5 min during tournament days, clients poll for the latest cached results. Document your findings about ESPN rate limits and the chosen approach in specs/espn-sync.md before implementing.
