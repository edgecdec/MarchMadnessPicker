- **Add autosave for bracket picks**: Auto-save picks to the server after each pick change (debounce by ~2 seconds so rapid clicks don't spam the API). Show a subtle status indicator next to the Save button: "✓ Saved" after successful autosave, "Saving..." during save, "⚠ Unsaved" if autosave fails. Keep the manual Save button as a fallback but make it clear that picks are already being auto-saved. This prevents users from losing picks if they forget to click Save or navigate away.

- **First Four scoring logic is wrong**: Users do NOT pick a specific First Four team. They pick "the winner of this First Four game" to advance. Example: R64 slot shows "NC State/Texas" vs BYU. User clicks "NC State/Texas" to beat BYU. If Texas wins the First Four and then beats BYU, the user gets credit — because they picked the First Four winner to advance, not a specific team. In the bracket, store First Four picks as the slot identifier (e.g. "ff-west-11") not a team name. For display, show "NC State/Texas" as a combined label. When the First Four resolves, replace the display with the actual winner but keep scoring the pick as correct if the First Four winner matches the user's advancement pick. The user never picks WHICH First Four team wins — they only pick whether that slot advances past the next opponent.
