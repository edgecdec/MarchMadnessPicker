# Bugs — Fix these BEFORE working on any PLAN.md tasks

- **First Four slots show only one team instead of both**: For unresolved First Four games, the R64 bracket slot should show "TeamA/TeamB" (e.g. "NC State/Texas") but currently it just shows one of the two teams. The bracket_data has a `first_four` array with `teamA` and `teamB` for each play-in game. The bracket UI needs to check if a First Four game feeding into an R64 slot is unresolved (no winner yet in results_data) and display both team names separated by a slash. Once the First Four game is resolved, show only the winner.

- **Bracket connector lines don't properly connect rounds**: The lines between matchup boxes across rounds are misaligned and don't visually connect the feeder games to the next round. Either fix the connector line CSS so they properly bridge between rounds, or remove them entirely if they can't be fixed cleanly. A bracket without connectors looks better than one with broken connectors.

- **Add autosave for bracket picks**: Auto-save picks to the server after each pick change (debounce by ~2 seconds so rapid clicks don't spam the API). Show a subtle status indicator next to the Save button: "✓ Saved" after successful autosave, "Saving..." during save, "⚠ Unsaved" if autosave fails. Keep the manual Save button as a fallback but make it clear that picks are already being auto-saved. This prevents users from losing picks if they forget to click Save or navigate away.

- **First Four pick logic clarification**: Users do NOT pick First Four games separately. The R64 slot should show "TeamA/TeamB" (e.g. "NC State/Texas") and the user picks whichever team they think will win the R64 game against the opposing seed. Clicking either team name advances that team through the bracket as their R64 pick. There should be no separate First Four picking UI — it's all handled inline in the R64 matchup display. The user is effectively picking both the First Four winner AND the R64 winner in one click.
