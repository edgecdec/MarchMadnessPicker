# Bugs — Fix these BEFORE working on any PLAN.md tasks


- **Refactor pick identification to use region+seed instead of team name strings**: Currently picks are stored and compared as team name strings (e.g. "Duke", "NC State/Texas"). This is fragile — name mismatches, First Four combined labels, and ESPN name differences all cause scoring bugs. Instead, identify picks by region+seed (e.g. "East-1", "West-11"). This is a unique identifier within a tournament (each region has exactly one team per seed). For First Four teams, both play-in teams share the same region+seed (e.g. "West-11" for both NC State and Texas), which is exactly what we want — the user is picking "the 11-seed from the West" to advance, not a specific team. Changes needed: (1) Store picks as region+seed values instead of team names in the picks object. (2) Store results as region+seed values. (3) Update scoring logic in src/lib/scoring.ts to compare region+seed. (4) Update the bracket UI to resolve region+seed back to team name for display. (5) Update leaderboard, distribution, simulator, and all other places that compare picks to results. This is a significant refactor but fixes the root cause of multiple bugs (First Four scoring, name mismatches, string comparison fragility).
